### 5.3.2 需求分析原则

1. 必须能够表示和理解问题的信息域
2. 必须能够定义将完成的任务
3. 必须能够表示软件的行为
4. 必须划分描述数据、功能和行为的模型，从而可以分层次地揭示细节
5. 分析过程应该从要素信息移向细节信息
